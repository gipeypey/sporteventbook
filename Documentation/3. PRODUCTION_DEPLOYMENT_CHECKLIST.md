# Production Deployment Checklist

## SportEventBook - NKP Production Deployment

Checklist ini digunakan untuk memastikan semua langkah deployment sudah dilakukan dengan benar.

---

## 📋 Pre-Deployment Checklist

### Infrastructure Preparation

- [ ] **NDB MySQL Setup**
  - [ ] MySQL instance sudah di-provision di NDB
  - [ ] IP address: 192.168.2.61 (atau sesuai)
  - [ ] Database `sporteventbook` sudah dibuat
  - [ ] User `sporteventbook` sudah dibuat dengan proper grants
  - [ ] Connection test berhasil dari bastion
  - [ ] Firewall rules sudah dibuka (port 3306)
  - [ ] Backup schedule sudah dikonfigurasi

- [ ] **NUS Object Storage Setup**
  - [ ] Bucket `sporteventbook-assets` sudah dibuat
  - [ ] Service account sudah dibuat
  - [ ] Access Key ID sudah digenerate
  - [ ] Secret Access Key sudah digenerate
  - [ ] Bucket policy sudah dikonfigurasi
  - [ ] Upload/download test berhasil
  - [ ] CORS sudah dikonfigurasi (jika perlu)

- [ ] **NKP Cluster Access**
  - [ ] Kubeconfig sudah di-download
  - [ ] kubectl sudah terinstall
  - [ ] Connection test: `kubectl cluster-info`
  - [ ] Node status: `kubectl get nodes`

- [ ] **Harbor Registry**
  - [ ] Harbor accessible
  - [ ] Credentials ready
  - [ ] Project `sporteventbook` sudah dibuat
  - [ ] Docker login test berhasil

- [ ] **Domain & SSL**
  - [ ] Domain `sport.bercalab.my.id` sudah disiapkan
  - [ ] DNS record akan di-point ke LoadBalancer IP
  - [ ] SSL certificate ready (atau cert-manager configured)

### Application Preparation

- [ ] **Code Ready**
  - [ ] Semua code changes sudah di-test
  - [ ] Migration files sudah siap
  - [ ] No hardcoded credentials di code
  - [ ] `.env.example` sudah updated

- [ ] **Docker Image**
  - [ ] Dockerfile sudah updated
  - [ ] Build test lokal berhasil
  - [ ] Image size optimal (< 500MB)

### Credentials & Secrets

- [ ] **APP_KEY**
  - [ ] Generated: `docker run --rm php:8.3 php -r "echo 'base64:' . base64_encode(random_bytes(32));"`
  - [ ] Disimpan di secret

- [ ] **Database Credentials**
  - [ ] DB_PASSWORD ready
  - [ ] Disimpan di secret

- [ ] **NUS Credentials**
  - [ ] AWS_ACCESS_KEY_ID ready
  - [ ] AWS_SECRET_ACCESS_KEY ready
  - [ ] Disimpan di secret

- [ ] **Midtrans Credentials**
  - [ ] MIDTRANS_SERVER_KEY ready
  - [ ] MIDTRANS_CLIENT_KEY ready
  - [ ] Disimpan di secret

## 🚀 Deployment Checklist

### Step 1: Create Namespace

- [ ] Namespace `sporteventbook` dibuat
- [ ] Context switch ke namespace: `kubectl config set-context --current --namespace=sporteventbook`

```bash
kubectl create namespace sporteventbook
```

### Step 2: Create Secrets

- [ ] app-secret created
- [ ] harbor-secret created

```bash
cd k8s/production
./deploy.sh v1.0.0
```

### Step 3: Deploy Infrastructure

- [ ] Redis deployed
- [ ] Redis pod running

```bash
kubectl get pods -n sporteventbook | grep redis
```

### Step 4: Build & Push Image

- [ ] Docker image built
- [ ] Image pushed to Harbor
- [ ] Image tag: v1.0.0

```bash
docker build -t harbor.your-domain.com/sporteventbook/app:v1.0.0 ../../
docker push harbor.your-domain.com/sporteventbook/app:v1.0.0
```

### Step 5: Deploy Application

- [ ] Laravel app deployment created
- [ ] Pods running (3 replicas)
- [ ] All pods in Running state

```bash
kubectl get pods -n sporteventbook | grep laravel
```

### Step 6: Deploy Nginx

- [ ] Nginx deployment created
- [ ] Nginx service created
- [ ] Pods running (2 replicas)

```bash
kubectl get pods -n sporteventbook | grep nginx
kubectl get svc -n sporteventbook | grep nginx
```

### Step 7: Deploy Ingress

- [ ] Ingress created
- [ ] Host: sport.bercalab.my.id
- [ ] TLS configured

```bash
kubectl get ingress -n sporteventbook
kubectl describe ingress sporteventbook-ingress -n sporteventbook
```

### Step 8: Run Migrations

- [ ] Migration job submitted
- [ ] Migration completed successfully

```bash
kubectl wait --for=condition=complete job/migrate -n sporteventbook --timeout=300s
kubectl logs job/migrate -n sporteventbook
```

### Step 9: Deploy Background Workers

- [ ] Queue worker deployed
- [ ] Scheduler (CronJob) deployed

```bash
kubectl get pods -n sporteventbook | grep queue
kubectl get cronjob -n sporteventbook
```

## ✅ Post-Deployment Verification

### Application Health

- [ ] **Pod Status**
  ```bash
  kubectl get pods -n sporteventbook
  ```
  Expected: All pods Running

- [ ] **Service Status**
  ```bash
  kubectl get svc -n sporteventbook
  ```
  Expected: nginx-service available

- [ ] **Ingress Status**
  ```bash
  kubectl get ingress -n sporteventbook
  ```
  Expected: ADDRESS populated

### Database Connection

- [ ] Test connection dari pod:
  ```bash
  kubectl run db-test --image=mysql:8.0 --rm -it -n sporteventbook \
      --env="MYSQL_PWD=<password>" \
      -- mysql -h 192.168.2.61 -u sporteventbook -D sporteventbook -e "SELECT 1;"
  ```

### Application Access

- [ ] **Homepage Access**
  - [ ] https://sport.bercalab.my.id accessible
  - [ ] No error pages
  - [ ] Assets (CSS/JS) loaded

- [ ] **Admin Panel**
  - [ ] https://sport.bercalab.my.id/admin accessible
  - [ ] Login form appears
  - [ ] Can login with credentials

- [ ] **API Test**
  - [ ] https://sport.bercalab.my.id/api/health returns 200

### Storage Test

- [ ] **S3 Upload Test**
  - [ ] Upload gambar event berhasil
  - [ ] Image tersimpan di NUS
  - [ ] Image bisa diakses

### Queue Test

- [ ] **Queue Worker**
  - [ ] Worker logs show no errors
  - [ ] Jobs processed successfully

```bash
kubectl logs -f deployment/queue-worker -n sporteventbook
```

### Scheduler Test

- [ ] **CronJob**
  - [ ] Scheduler running setiap menit
  - [ ] Logs show successful execution

```bash
kubectl get cronjob -n sporteventbook
kubectl get jobs -n sporteventbook
```
