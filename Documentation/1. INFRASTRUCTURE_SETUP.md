# Infrastructure Setup Guide

## SportEventBook - Nutanix Platform Deployment

Dokumentasi ini menjelaskan setup infrastruktur lengkap untuk deployment SportEventBook ke Nutanix Kubernetes Platform (NKP) dengan:
- **NDB MySQL** - Database as a Service
- **NUS Object Storage** - S3-compatible storage untuk file uploads
- **Traefik Ingress** - Load balancing dan routing
- **MetalLB** - LoadBalancer implementation

## 🗄️ Part 1: NDB MySQL Setup

### 1.1 Akses NDB Console

1. Buka browser dan akses NDB Console
   ```
   https://<ndb-server-ip>:9440
   ```

2. Login dengan credentials Anda
   - Username: `era`
   - Password: `P@ssw0rd` (sesuaikan dengan environment Anda)

### 1.2 Provision MySQL Database

1. **Navigate ke Databases**
   - Klik menu **Databases** di sidebar kiri
   - Klik tombol **Provision a Database**

2. **Pilih Database Type**
   - Pilih **MySQL**
   - Pilih versi: **MySQL 8.0** atau lebih baru

3. **Configure Database**
   ```
   ┌─────────────────────────────────────────────────┐
   │ Database Configuration                          │
   ├─────────────────────────────────────────────────┤
   │ Name:           sporteventbook                  │
   │ Description:    SportEventBook Production DB    │
   │ Version:        MySQL 8.0.x                     │
   │ Database Name:  sporteventbook                  │
   │ Username:       root                            │
   │ Password:       [Set secure password]           │
   │ Confirm:        [Confirm password]              │
   └─────────────────────────────────────────────────┘
   ```

4. **Configure Compute & Storage**
   ```
   ┌─────────────────────────────────────────────────┐
   │ Compute & Storage Configuration                 │
   ├─────────────────────────────────────────────────┤
   │ vCPU:             4                             │
   │ Memory:           8 GB                          │
   │ Storage:          50 GB                         │
   │ Storage Type:     SSD                           │
   │ Replication:      Single Instance (atau HA)     │
   └─────────────────────────────────────────────────┘
   ```

5. **Network Configuration**
   ```
   ┌─────────────────────────────────────────────────┐
   │ Network Configuration                           │
   ├─────────────────────────────────────────────────┤
   │ VLAN:           [Select appropriate VLAN]       │
   │ IP Assignment:  Static                          │
   │ IP Address:     192.168.2.61                    │
   │ Subnet Mask:    255.255.255.0                   │
   │ Gateway:        192.168.2.1                     │
   │ DNS:            8.8.8.8, 8.8.4.4                │
   └─────────────────────────────────────────────────┘
   ```

6. **Review dan Provision**
   - Review semua konfigurasi
   - Klik **Provision**
   - Tunggu proses provisioning selesai (~10-15 menit)

### 1.3 Verifikasi Database Connection

1. **Dari Laptop/Local**
   ```bash
   # Install MySQL CLI jika belum
   # Windows: download dari mysql.com
   # Mac: brew install mysql
   # Linux: sudo apt install mysql-client

   # Test connection
   mysql -h 192.168.2.61 -u root -p

   # Enter password saat diminta
   ```

2. **Dari VM Bastion**
   ```bash
   # SSH ke bastion
   ssh -i ~/.ssh/id_rsa user@bastion-ip

   # Test connection
   mysql -h 192.168.2.61 -u root -p
   ```

3. **Create Database & User untuk Aplikasi**
   ```sql
   -- Login sebagai root
   mysql -h 192.168.2.61 -u root -p

   -- Create database
   CREATE DATABASE IF NOT EXISTS sporteventbook 
   CHARACTER SET utf8mb4 
   COLLATE utf8mb4_unicode_ci;

   -- Create user untuk aplikasi
   CREATE USER 'sporteventbook'@'%' IDENTIFIED BY 'YourSecurePassword123!';

   -- Grant privileges
   GRANT ALL PRIVILEGES ON sporteventbook.* TO 'sporteventbook'@'%';

   -- Flush privileges
   FLUSH PRIVILEGES;

   -- Verify
   SHOW DATABASES;
   SHOW GRANTS FOR 'sporteventbook'@'%';

   -- Exit
   EXIT;
   ```

### 1.4 Configure Firewall Rules

Pastikan port 3306 terbuka dari Kubernetes cluster ke NDB MySQL:

```bash
# Dari NKP Console atau CLI
# Allow traffic dari Kubernetes pod CIDR ke MySQL port

# Contoh jika menggunakan iptables di NDB VM
sudo iptables -A INPUT -p tcp -s 10.0.0.0/8 --dport 3306 -j ACCEPT
sudo iptables-save > /etc/iptables/rules.v4
```

### 1.5 Backup Configuration (NDB)

NDB menyediakan automated backup:

1. **Setup Backup Schedule**
   - Navigate ke database di NDB Console
   - Klik **Backup** tab
   - Klik **Configure Backup**
   - Set schedule: Daily, 02:00 AM
   - Retention: 7 days

2. **Manual Backup**
   ```bash
   # Via NDB CLI atau API
   ndb_cli database backup --name sporteventbook --description "Pre-deployment backup"
   ```

## 🪣 Part 2: NUS Object Storage Setup

### 2.1 Akses NUS Console

1. Buka browser dan akses NUS Console
   ```
   https://<nus-server-ip>:8080
   ```
   atau via NKP Console → Storage → Object Storage

### 2.2 Create Bucket

1. **Navigate ke Buckets**
   - Klik menu **Buckets** di sidebar
   - Klik **Create Bucket**

2. **Configure Bucket**
   ```
   ┌─────────────────────────────────────────────────┐
   │ Bucket Configuration                            │
   ├─────────────────────────────────────────────────┤
   │ Bucket Name:    sporteventbook-assets           │
   │ Region:         local (atau sesuai setup)       │
   │ Versioning:     Enabled                         │
   │ Access:         Private                         │
   │ Quota:          100 GB                          │
   └─────────────────────────────────────────────────┘
   ```

3. **Create Bucket**
   - Klik **Create Bucket**
   - Bucket akan dibuat dan siap digunakan

### 2.3 Create Access Credentials

1. **Navigate ke Access Management**
   - Klik menu **Access Management** atau **IAM**
   - Klik **Create Access Key**

2. **Create Service Account**
   ```
   ┌─────────────────────────────────────────────────┐
   │ Service Account Configuration                   │
   ├─────────────────────────────────────────────────┤
   │ Account Name:   sporteventbook-app              │
   │ Description:    App access to object storage    │
   │ Permissions:    Read/Write on sporteventbook-assets │
   └─────────────────────────────────────────────────┘
   ```

3. **Generate Access Keys**
   - Setelah create account, generate access keys
   - **Simpan credentials ini** (hanya muncul sekali!)
   ```
   Access Key ID:     AKIAIOSFODNN7EXAMPLE
   Secret Access Key: wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
   ```

4. **Set Bucket Policy**
   ```json
   {
     "Version": "2012-10-17",
     "Statement": [
       {
         "Effect": "Allow",
         "Principal": {
           "AWS": "arn:aws:iam:::user/sporteventbook-app"
         },
         "Action": [
           "s3:GetObject",
           "s3:PutObject",
           "s3:DeleteObject"
         ],
         "Resource": [
           "arn:aws:s3:::sporteventbook-assets/*"
         ]
       },
       {
         "Effect": "Allow",
         "Principal": {
           "AWS": "arn:aws:iam:::user/sporteventbook-app"
         },
         "Action": [
           "s3:ListBucket"
         ],
         "Resource": [
           "arn:aws:s3:::sporteventbook-assets"
         ]
       }
     ]
   }
   ```

### 2.4 Test Object Storage Connection

1. **Install AWS CLI (Optional)**
   ```bash
   # Download dari https://aws.amazon.com/cli/
   # Atau via package manager
   brew install awscli  # Mac
   sudo apt install awscli  # Linux
   ```

2. **Configure AWS CLI**
   ```bash
   aws configure
   # AWS Access Key ID: AKIAIOSFODNN7EXAMPLE
   # AWS Secret Access Key: wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
   # Default region name: us-east-1
   # Default output format: json
   ```

3. **Test Upload/Download**
   ```bash
   # Set endpoint URL untuk NUS
   export AWS_ENDPOINT_URL=http://<nus-server-ip>:8080

   # List buckets
   aws s3 ls --endpoint-url http://<nus-server-ip>:8080

   # Upload test file
   echo "test" > test.txt
   aws s3 cp test.txt s3://sporteventbook-assets/test.txt --endpoint-url http://<nus-server-ip>:8080

   # Download test file
   aws s3 cp s3://sporteventbook-assets/test.txt ./test-downloaded.txt --endpoint-url http://<nus-server-ip>:8080

   # List objects in bucket
   aws s3 ls s3://sporteventbook-assets --endpoint-url http://<nus-server-ip>:8080
   ```

### 2.5 Configure CORS (Untuk Frontend Upload)

Jika frontend perlu upload langsung ke object storage:

```json
{
  "CORSRules": [
    {
      "AllowedHeaders": ["*"],
      "AllowedMethods": ["GET", "PUT", "POST", "DELETE"],
      "AllowedOrigins": ["https://sport.bercalab.my.id"],
      "ExposeHeaders": ["ETag"],
      "MaxAgeSeconds": 3600
    }
  ]
}
```

Apply CORS via AWS CLI:
```bash
aws s3api put-bucket-cors \
    --bucket sporteventbook-assets \
    --cors-configuration file://cors.json \
    --endpoint-url http://<nus-server-ip>:8080
```

## 🔧 Part 3: Update Aplikasi Configuration

### 3.1 Update .env File

Update file `.env` di project Anda:

```bash
# .env
APP_NAME=SportEventBook
APP_ENV=production
APP_DEBUG=false
APP_URL=https://sport.bercalab.my.id

# Database - NDB MySQL
DB_CONNECTION=mysql
DB_HOST=192.168.2.61
DB_PORT=3306
DB_DATABASE=sporteventbook
DB_USERNAME=sporteventbook
DB_PASSWORD=YourSecurePassword123!

# Cache & Queue - Redis
CACHE_DRIVER=redis
QUEUE_CONNECTION=redis
SESSION_DRIVER=redis
REDIS_HOST=redis
REDIS_PORT=6379

# Object Storage - NUS
FILESYSTEM_DRIVER=s3
AWS_ACCESS_KEY_ID=AKIAIOSFODNN7EXAMPLE
AWS_SECRET_ACCESS_KEY=wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
AWS_DEFAULT_REGION=us-east-1
AWS_BUCKET=sporteventbook-assets
AWS_URL=http://<nus-server-ip>:8080/sporteventbook-assets
AWS_ENDPOINT=http://<nus-server-ip>:8080

# Midtrans
MIDTRANS_SERVER_KEY=SB-Mid-server-xxxxx
MIDTRANS_CLIENT_KEY=SB-Mid-client-xxxxx
MIDTRANS_IS_PRODUCTION=false
```

### 3.2 Update filesystems.php

Update `config/filesystems.php` untuk menggunakan S3:

```php
// config/filesystems.php
return [
    // ...
    
    'disks' => [
        // ...
        
        'public' => [
            'driver' => 'local',
            'root' => storage_path('app/public'),
            'url' => env('APP_URL').'/storage',
            'visibility' => 'public',
        ],

        's3' => [
            'driver' => 's3',
            'key' => env('AWS_ACCESS_KEY_ID'),
            'secret' => env('AWS_SECRET_ACCESS_KEY'),
            'region' => env('AWS_DEFAULT_REGION'),
            'bucket' => env('AWS_BUCKET'),
            'url' => env('AWS_URL'),
            'endpoint' => env('AWS_ENDPOINT'),
            'use_path_style_endpoint' => true,
            'throw' => false,
        ],
    ],

    // ...
];
```

### 3.3 Update Model untuk Upload ke S3

Untuk upload gambar event ke S3, update model atau controller:

```php
// App/Models/Event.php atau EventController.php
use Illuminate\Support\Facades\Storage;

// Upload gambar
if ($request->hasFile('image')) {
    $image = $request->file('image');
    $path = $image->store('events', 's3'); // Store di S3
    
    $event->image_path = $path;
    $event->save();
}
```

## 🚀 Part 4: Kubernetes Deployment

### 4.1 Update Kubernetes Manifests

File manifests yang perlu diupdate:

1. **k8s/app-secret.yaml** - Tambah credentials NUS
2. **k8s/app-configmap.yaml** - Update DB_HOST ke NDB IP
3. **k8s/app-deployment.yaml** - Hapus storage PVC (gunakan S3)

### 4.2 Create/Update Secrets

```bash
# Namespace
kubectl create namespace sporteventbook --dry-run=client -o yaml | kubectl apply -f -

# App Secret dengan NDB + NUS credentials
kubectl create secret generic app-secret \
    --from-literal=APP_KEY="base64:YourAppKey32Characters==" \
    --from-literal=DB_HOST="192.168.2.61" \
    --from-literal=DB_PORT="3306" \
    --from-literal=DB_DATABASE="sporteventbook" \
    --from-literal=DB_USERNAME="sporteventbook" \
    --from-literal=DB_PASSWORD="YourSecurePassword123!" \
    --from-literal=AWS_ACCESS_KEY_ID="AKIAIOSFODNN7EXAMPLE" \
    --from-literal=AWS_SECRET_ACCESS_KEY="wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY" \
    --from-literal=MIDTRANS_SERVER_KEY="SB-Mid-server-xxxxx" \
    --from-literal=MIDTRANS_CLIENT_KEY="SB-Mid-client-xxxxx" \
    -n sporteventbook \
    --dry-run=client -o yaml | kubectl apply -f -

# Harbor Secret
kubectl create secret docker-registry harbor-secret \
    --docker-server=harbor.your-domain.com \
    --docker-username=admin \
    --docker-password='YourHarborPassword' \
    --docker-email=admin@bercalab.my.id \
    -n sporteventbook \
    --dry-run=client -o yaml | kubectl apply -f -
```

### 4.3 Update ConfigMap

```bash
kubectl create configmap app-config \
    --from-literal=APP_NAME="SportEventBook" \
    --from-literal=APP_ENV="production" \
    --from-literal=APP_DEBUG="false" \
    --from-literal=APP_URL="https://sport.bercalab.my.id" \
    --from-literal=DB_CONNECTION="mysql" \
    --from-literal=CACHE_DRIVER="redis" \
    --from-literal=QUEUE_CONNECTION="redis" \
    --from-literal=SESSION_DRIVER="redis" \
    --from-literal=REDIS_HOST="redis" \
    --from-literal=REDIS_PORT="6379" \
    --from-literal=FILESYSTEM_DRIVER="s3" \
    --from-literal=LOG_CHANNEL="stderr" \
    --from-literal=LOG_LEVEL="info" \
    -n sporteventbook \
    --dry-run=client -o yaml | kubectl apply -f -
```

### 4.4 Deploy Redis

```bash
kubectl apply -f k8s/redis-deployment.yaml -n sporteventbook
```

### 4.5 Build & Push Docker Image

```bash
# Login Harbor
docker login harbor.your-domain.com

# Build
docker build -t harbor.your-domain.com/sporteventbook/app:v1.0.0 .

# Push
docker push harbor.your-domain.com/sporteventbook/app:v1.0.0
```

### 4.6 Deploy Application

```bash
kubectl apply -f k8s/app-deployment.yaml -n sporteventbook
```

### 4.7 Run Migrations

```bash
kubectl apply -f k8s/migration-job.yaml -n sporteventbook
kubectl wait --for=condition=complete job/migrate -n sporteventbook --timeout=300s
```

### 4.8 Deploy Queue Worker & Scheduler

```bash
kubectl apply -f k8s/queue-worker-deployment.yaml -n sporteventbook
kubectl apply -f k8s/scheduler-cronjob.yaml -n sporteventbook
```

### 4.9 Setup Ingress (Traefik)

Create `k8s/ingress.yaml`:

```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: sporteventbook-ingress
  namespace: sporteventbook
  annotations:
    kubernetes.io/ingress.class: traefik
    traefik.ingress.kubernetes.io/router.entrypoints: websecure
    cert-manager.io/cluster-issuer: letsencrypt-prod
spec:
  rules:
  - host: sport.bercalab.my.id
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: nginx-service
            port:
              number: 80
  tls:
  - hosts:
    - sport.bercalab.my.id
    secretName: sporteventbook-tls
```

Apply:
```bash
kubectl apply -f k8s/ingress.yaml -n sporteventbook
```

## 🔍 Troubleshooting

### Cannot connect to NDB MySQL
```bash
# Check firewall
telnet 192.168.2.61 3306

# Check MySQL service
kubectl run mysql-test --image=mysql:8.0 --rm -it -n sporteventbook \
    --env="MYSQL_PWD=YourPassword" \
    -- mysql -h 192.168.2.61 -u sporteventbook -e "SELECT 1;"
```

### S3 Upload Fails
```bash
# Check credentials
aws s3 ls s3://sporteventbook-assets \
    --access-key AKIAIOSFODNN7EXAMPLE \
    --secret-key wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY \
    --endpoint-url http://<nus-server-ip>:8080

# Check bucket policy
aws s3api get-bucket-policy \
    --bucket sporteventbook-assets \
    --endpoint-url http://<nus-server-ip>:8080
```

### Pod Cannot Access External Services
```bash
# Check network policies
kubectl get networkpolicies -n sporteventbook

# Test DNS resolution
kubectl run dns-test --image=busybox --rm -it -n sporteventbook -- \
    nslookup 192.168.2.61
```
